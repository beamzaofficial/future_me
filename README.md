# Future Me — Time Capsule Letters

A cross-platform Flutter app where you write letters to your future self. A
letter is **sealed** until the unlock date you choose — only then can you (or
the public, if you opted to share) read it. Built with Firebase Authentication
and Cloud Firestore for the SEA606 term project.

> **Theme:** time-released letters with a public wall
> **Platforms:** Android + Web
> **Auth:** Google Sign-In
> **Cloud data:** Cloud Firestore (`letters` collection, full CRUD)
> **Local data:** SharedPreferences (theme, notification preference, draft auto-save, last-vault-viewed timestamp)

---

## 1. Quick start (after Firebase is configured)

```bash
flutter pub get
flutter run -d chrome      # web
flutter run -d <android>   # Android device or emulator
```

If you have not yet configured Firebase, see **Section 3 — Firebase setup**.

## 3. Firebase setup (one-time)

The project ships with a placeholder `lib/firebase_options.dart` that throws a
clear error until you configure your own Firebase project. Steps:

1. **Create a Firebase project** at https://console.firebase.google.com.
2. **Enable Authentication → Google** as a sign-in method. Set a project
   support email.
3. **Create a Cloud Firestore database** in production mode (region: anywhere
   close to you). The rules below will lock it down.
4. **Install the FlutterFire CLI** (one-time per machine):
   ```bash
   dart pub global activate flutterfire_cli
   ```
5. **Generate `firebase_options.dart`** for both platforms:
   ```bash
   flutterfire configure --platforms=android,web
   ```
   Pick your project, accept the suggested Android package
   `com.example.future_me`, and accept the auto-generated web app.
6. **Deploy Firestore rules and indexes**:
   ```bash
   # Once: install Firebase CLI and login
   npm install -g firebase-tools
   firebase login
   firebase use --add        # select your project
   # Then deploy
   firebase deploy --only firestore:rules,firestore:indexes
   ```

### Web — Google Sign-In

The web app signs in through `FirebaseAuth.signInWithPopup`, so it does not
need a `google-signin-client_id` meta tag. In Firebase Console, make sure the
Google provider is enabled and your local/deployed host is listed under
Authentication → Settings → Authorized domains.

### Android — SHA-1 fingerprint

For Google Sign-In on Android, add your debug SHA-1 to the Firebase project:

```bash
cd android && ./gradlew signingReport
```

Copy the `SHA1` line from the `debug` variant and paste it into
Firebase Console → Project Settings → General → Your Android app → Add fingerprint.
Then re-run `flutterfire configure` so `google-services.json` is updated.

## 5. Running tests

```bash
# All unit + widget tests (no device needed)
flutter test

# Just unit
flutter test test/unit/

# Just widget
flutter test test/widget/

# Integration tests — requires a device or browser
flutter test integration_test/app_flow_test.dart -d chrome
flutter test integration_test/app_flow_test.dart -d <android_device_id>
```

The integration test wires `FakeFirebaseFirestore` and a stub `AuthService`
into the real app entry point, so it runs without touching your real Firebase
project.

## 8. Notes & known limits

* The integration test bypasses the date picker (platform-specific UI) by
  pre-seeding letters into `FakeFirebaseFirestore`. The write form is exercised
  by `test/widget/write_page_test.dart` (validation + draft auto-save) and by
  manual testing on each platform.
* Notification preferences are stored locally but the actual local-notification
  delivery is intentionally out of scope (would require `flutter_local_notifications`
  and platform permissions).
* The countdown badge ticks once per second using a `Timer.periodic`. For a
  long list of locked letters, this is fine on web/Android but could be made
  per-screen if the list grows large.
